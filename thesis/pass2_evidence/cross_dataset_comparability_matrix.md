## 2. Final Comparison Frame

The thesis compares prediction methods across two RTS titles under **structurally different data regimes**:

- **SC2EGSet:** Professional/tournament replay corpus with replay-derived in-game telemetry. Population: professional/tournament players (~2,495 distinct identities). Feature space: full pre-game + in-game trajectory extracted from three event streams. Rating: Battle.net MMR structurally absent for 83.95% of rows (unrated professional); Glicko-2 reconstructable from history.

- **aoestats:** Source-specific 1v1 Random Map leaderboard records with semantic opacity around queue type (Tier 4). Population: public AoE2 DE players appearing under the `leaderboard='random_map'` aggregation label at aoestats.io; queue type (ranked vs quickplay vs custom) unverifiable from available documentation. Feature space: pre-game only (rating, civ, map, opponent civ, matchup). Do NOT call simply "ranked ladder".

- **aoe2companion:** Public 1v1 Random Map **mixed-mode** records spanning two leaderboards. **Leaderboard-activity distribution** (`leaderboard_raw` table): `rm_1v1` (ID 6) ≈ 54M raw entries (~88.5% within the lb=6+18 subset; ranked candidate); `qp_rm_1v1` (ID 18) ≈ 7M raw entries (~11.5%; quickplay/matchmaking-derived per T05 fallback rule). **Post-cleaning analytical view** (`matches_1v1_clean`): 30,531,196 match-pairs / 61,062,392 player-rows / 683,790 distinct `profileId`s in the lb=6+18 cohort (the leaderboard_raw activity counts and the post-cleaning match-pair count are at different grains and must not be conflated). Population: public AoE2 DE players across ranked and quickplay/matchmaking modes. Feature space: pre-game only (rating, civ, map, opponent civ, matchup). Do NOT call simply "ranked ladder" for the combined scope (ID 18 is Tier 3 quickplay per T05).

**Observed differences between SC2 and AoE2 results MUST be interpreted as differences in method, data regime, and population — NOT as pure game-mechanic differences.** The SC2EGSet professional replay corpus and the AoE2 public leaderboard/matchmaking records represent structurally incomparable populations and feature availability profiles. Any cross-game comparison is conditional on this asymmetry being acknowledged.

### Five explicit comparison axes

The matrix explicitly distinguishes these five axes:

1. **Game comparison:** StarCraft II vs Age of Empires II as RTS titles. This is the framing axis only — it does not imply all differences are game-mechanic in origin.

2. **Data-regime comparison:** SC2 professional tournament replay telemetry (full in-game event streams, 22K matches, 46 patch versions, tournament-structured) vs AoE2 public third-party random-map records (no replay telemetry, 17.8M–30.5M matches, community-aggregated API snapshots).

3. **Population comparison:** Professional/tournament population (SC2EGSet, ~2,495 players, Heckman selection bias toward high skill) vs public leaderboard/matchmaking population (aoestats post-cleaning view: ~641K `profile_id`s; aoe2companion post-cleaning lb=6+18 cohort: ~683K `profileId`s — distinct from the 3.4M cross-leaderboard `profileId` cardinality in `matches_raw`, which spans all leaderboards at a different grain; ICC near-zero in both post-cleaning cohorts, indicating low per-player signal in the reference window).

4. **Feature-availability comparison:** SC2 replay-derived in-game state (three event streams, economic/military/technological trajectories, ~7s granularity) vs AoE2 mostly pre-game/history/rating/map/civilization features (no in-game state; `opening` + age_uptimes available only pre-2024-03-10 in aoestats and absent in aoe2companion).

5. **Inference limitation:** Results cannot be interpreted as a clean causal statement about game mechanics alone. Observed differences in model accuracy, feature importance, or method rankings between SC2 and AoE2 may reflect any combination of: (a) differences in underlying game mechanics; (b) differences in data regime (replay telemetry vs aggregated statistics); (c) differences in population (professional vs public); (d) differences in feature availability (full in-game vs pre-game only). No design in this thesis allows these four sources to be cleanly separated. All cross-game comparison claims must acknowledge this limitation.
